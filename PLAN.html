star_crawl — universal web crawler with content extraction

Một crawler đa nguồn, output LLM-ready markdown / parquet, targeting Uber · Grab · Gojek · Medium và tech blog ecosystem. Plug-in architecture: thêm source mới = thêm 1 file YAML.

Owner — @baotran8810 Lang — Python 3.11+ License — MIT (TBC) Status — awaiting confirmation

Mục tiêu & Scope

Crawler phải xử lý được nhiều profile site khác nhau: SSR, JS-rendered, Cloudflare-fronted, paywalled. Output sạch, dedup, có metadata.

Crawl đa nguồn

Báo, blog, tech publication. Static HTML + JS-heavy + RSS + sitemap.

Lọc content sạch

Bỏ nav/ads/footer/boilerplate. Giữ main content + metadata (title, author, date, lang).

LLM-ready

Markdown + JSON có schema, sẵn sàng cho RAG / fine-tune dataset.

Pluggable

Thêm source = thêm 1 YAML. Không sửa core code.

Polite

Tôn trọng robots.txt, rate-limit, retry, dedup.

Resumable

SQLite frontier, restart không mất tiến độ.

Source configs (declarative)

Add source mới = thêm 1 file YAML. Không sửa core code.

Uber — pagination + Cloudflare-friendly

# configs/sources/uber_engineering.yaml
name: uber_engineering
fetcher: http
seed:
  strategy: pagination
  template: "https://www.uber.com/us/en/blog/engineering/page/{n}/"
  range: [1, 63]
url_filter: '^https://www\.uber\.com/us/en/blog/[^/]+/$'
rate_limit: { rps: 1, concurrency: 5 }
extract:
  parse_jsonld: true        # author, datePublished từ <script type="application/ld+json">

Grab — RSS, dễ nhất

# configs/sources/grab.yaml
name: grab_engineering
fetcher: http
seed:
  strategy: rss
  url: "https://engineering.grab.com/feed.xml"
url_filter: '^https://engineering\.grab\.com/[^/]+$'
rate_limit: { rps: 2, concurrency: 5 }

Gojek — browser (Vercel challenge)

# configs/sources/gojek.yaml
name: gojek_engineering
fetcher: browser
seed:
  strategy: pagination
  template: "https://www.gojek.io/blog?page={n}"
  range: [1, 99]              # auto-stop khi không còn link mới
url_filter: '^https://www\.gojek\.io/blog/[^/]+'
rate_limit: { rps: 0.5, concurrency: 2 }
browser:
  stealth: true
  wait_until: networkidle

Medium adapter (opt-in, experimental)

Medium robots.txt declare rõ User-Agent: ClaudeBot Disallow: /. Crawl chỉ khi user opt-in flag và hiểu rõ ToS.

Lưu ý ToS & pháp lý: Cả 3 phương án dưới cùng share rủi ro — Medium ToS prohibit API scraping; robots.txt explicit cấm AI crawlers. Code Freedium MIT-licensed nhưng không miễn trách nhiệm về content mà nó parse. Personal research & RAG cá nhân: rủi ro thấp. Redistribute / commercial training: rủi ro thật, không khuyến nghị.
#ApproachEffortReliabilityMaintenance
A Vendor medium-parser + gọi Medium GraphQL trực tiếp đề xuất 1–1.5d Cao (mình kiểm soát) TB — Medium đổi API là gãy
B Self-host Freedium docker-compose min profile 1.5d Cao Cao — 4 container + WARP
C Route URL qua freedium-mirror.cfd runtime 0.3d Thấp (mirror downtime) Thấp nhưng brittle

Adapter design (Option A)

# configs/sources/medium_graphql.yaml
name: medium_graphql
fetcher: medium_graphql            # custom adapter, vendored from Freedium
seed:
  strategy: hybrid
  feeds:                           # public feeds for discovery
    - "https://medium.com/feed/tag/distributed-systems"
    - "https://medium.com/feed/tag/machine-learning"
  follow_member_only: true
url_filter: '^https://medium\.com/.*-[a-f0-9]{8,}$'
extract:
  via_graphql: true                # post-id từ URL → GraphQL → HTML → trafilatura
  fallback_to_rss_content: true    # nếu GraphQL fail, dùng <content:encoded>
network:
  warp_proxy: optional             # cloudflare/cloudflared WARP
  rate_limit: { rps: 0.3, concurrency: 1 }
guards:
  require_flag: --i-understand-medium-tos    # default OFF

Risks & mitigations

Crawler dài hạn — phải tính đến site đổi DOM, anti-bot upgrade, ToS thay đổi.

RiskLikelihoodMitigation
Vercel siết bot challenge → Gojek block playwrightTBThêm curl-impersonate / cycletls fallback
Cloudflare Uber siết __cf_bmThấpThrottle 0.5 rps, rotate UA, dùng cookie session từ playwright
Medium đổi GraphQL schemaCaoPin medium-parser version, snapshot test, theo dõi Freedium upstream
Site đổi DOMCaotrafilatura ổn định + readability fallback + selector hint trong YAML
IP bị ban khi bulk crawlTBPolite rate-limit, exponential backoff, optional WARP/proxy
Dataset chứa duplicateTBSHA-256 content_hash + MinHash near-dup ở P5

Open questions cần chốt

Trả lời 4 câu là scaffold P0 + P1 (Grab) ngay luôn.

  1. Mục đích cuối của dataset: research cá nhân / RAG riêng, hay training/sản phẩm thương mại? → quyết định Medium adapter có nên implement không.
  2. Có server Linux + Docker để chạy WARP proxy (cho Medium adapter) không?
  3. Output mặc định: JSONL + Markdown? Hay parquet luôn?
  4. Phạm vi P1: chỉ Grab (validate pipeline), hay scaffold cho cả Uber/Gojek song song?

Plan này được build dần qua nhiều bước verification thực tế (probe headers, robots.txt, repo inspect). Tất cả số liệu site đều đã check live tại 2026-05-09. Nguồn: crawl4ai · firecrawl · trafilatura benchmark · Freedium-cfd/web.
