star_crawl — universal web crawler with content extraction

Một crawler đa nguồn, lưu vào SQLite, có web UI để duyệt & search, output LLM-ready markdown / parquet, targeting Uber · Grab · Gojek · Medium và tech blog ecosystem. Plug-in architecture: thêm source mới = thêm 1 file YAML.

Owner — @baotran8810 Lang — Python 3.11+ License — MIT (TBC) Status — awaiting confirmation

Mục tiêu & Scope

Crawler phải xử lý được nhiều profile site khác nhau: SSR, JS-rendered, Cloudflare-fronted, paywalled. Output sạch, dedup, có metadata.

Crawl đa nguồn

Báo, blog, tech publication. Static HTML + JS-heavy + RSS + sitemap.

Lọc content sạch

Bỏ nav/ads/footer/boilerplate. Giữ main content + metadata (title, author, date, lang).

LLM-ready

Markdown + JSON có schema, sẵn sàng cho RAG / fine-tune dataset.

Pluggable

Thêm source = thêm 1 YAML. Không sửa core code.

Polite

Tôn trọng robots.txt, rate-limit, retry, dedup.

Resumable

SQLite frontier, restart không mất tiến độ.

Source configs (declarative)

Add source mới = thêm 1 file YAML. Không sửa core code.

Uber — pagination + Cloudflare-friendly

# configs/sources/uber_engineering.yaml
name: uber_engineering
fetcher: http
seed:
  strategy: pagination
  template: "https://www.uber.com/us/en/blog/engineering/page/{n}/"
  range: [1, 63]
url_filter: '^https://www\.uber\.com/us/en/blog/[^/]+/$'
rate_limit: { rps: 1, concurrency: 5 }
extract:
  parse_jsonld: true        # author, datePublished từ <script type="application/ld+json">

Grab — RSS, dễ nhất

# configs/sources/grab.yaml
name: grab_engineering
fetcher: http
seed:
  strategy: rss
  url: "https://engineering.grab.com/feed.xml"
url_filter: '^https://engineering\.grab\.com/[^/]+$'
rate_limit: { rps: 2, concurrency: 5 }

Gojek — browser (Vercel challenge)

# configs/sources/gojek.yaml
name: gojek_engineering
fetcher: browser
seed:
  strategy: pagination
  template: "https://www.gojek.io/blog?page={n}"
  range: [1, 99]              # auto-stop khi không còn link mới
url_filter: '^https://www\.gojek\.io/blog/[^/]+'
rate_limit: { rps: 0.5, concurrency: 2 }
browser:
  stealth: true
  wait_until: networkidle

Database schema (SQLite)

SQLite làm primary store cho article đã extract. WAL mode để web UI đọc song song với crawler ghi. FTS5 cho full-text search. JSONL/parquet export derive từ DB — single source of truth.

Tại sao SQLite? Single-file, không cần server, FTS5 built-in, WAL cho concurrent reads. Khối lượng dự kiến ~5-10k articles — SQLite handle thoải mái. Nếu sau scale ra Postgres chỉ cần đổi repository.py.

Tables

-- nguồn crawl, 1-1 với configs/sources/*.yaml
CREATE TABLE sources (
  id            INTEGER PRIMARY KEY,
  name          TEXT UNIQUE NOT NULL,        -- 'uber_engineering'
  display_name  TEXT NOT NULL,                -- 'Uber Engineering'
  homepage_url  TEXT NOT NULL,
  config_path   TEXT NOT NULL,                -- 'configs/sources/uber_engineering.yaml'
  enabled       INTEGER NOT NULL DEFAULT 1,
  created_at    TEXT NOT NULL DEFAULT (datetime('now'))
);

-- bài đã crawl + extract
CREATE TABLE articles (
  id                INTEGER PRIMARY KEY,
  source_id         INTEGER NOT NULL REFERENCES sources(id) ON DELETE CASCADE,
  url               TEXT UNIQUE NOT NULL,
  url_hash          TEXT NOT NULL,            -- SHA-256 dedup
  content_hash      TEXT,                     -- SHA-256 body — near-dup
  title             TEXT,
  author            TEXT,
  published_at      TEXT,                     -- ISO 8601
  language          TEXT,                     -- 'en', 'vi', ...
  excerpt           TEXT,
  content_markdown  TEXT NOT NULL,            -- LLM-ready
  content_html      TEXT,                     -- raw extracted, optional
  word_count        INTEGER,
  tags_json         TEXT,                     -- JSON array
  metadata_json     TEXT,                     -- catch-all (JSON-LD, og:*, ...)
  fetched_at        TEXT NOT NULL,
  extracted_at      TEXT NOT NULL,
  status            TEXT NOT NULL DEFAULT 'ok' -- 'ok' | 'failed' | 'skipped'
);

CREATE INDEX idx_articles_source_published
  ON articles(source_id, published_at DESC);
CREATE INDEX idx_articles_url_hash      ON articles(url_hash);
CREATE INDEX idx_articles_content_hash  ON articles(content_hash);

-- full-text search
CREATE VIRTUAL TABLE articles_fts USING fts5(
  title, excerpt, content_markdown,
  content='articles', content_rowid='id',
  tokenize='unicode61 remove_diacritics 2'
);

-- triggers để sync articles ↔ articles_fts
CREATE TRIGGER articles_ai AFTER INSERT ON articles BEGIN
  INSERT INTO articles_fts(rowid, title, excerpt, content_markdown)
  VALUES (new.id, new.title, new.excerpt, new.content_markdown);
END;
CREATE TRIGGER articles_ad AFTER DELETE ON articles BEGIN
  INSERT INTO articles_fts(articles_fts, rowid, title, excerpt, content_markdown)
  VALUES ('delete', old.id, old.title, old.excerpt, old.content_markdown);
END;
CREATE TRIGGER articles_au AFTER UPDATE ON articles BEGIN
  INSERT INTO articles_fts(articles_fts, rowid, title, excerpt, content_markdown)
  VALUES ('delete', old.id, old.title, old.excerpt, old.content_markdown);
  INSERT INTO articles_fts(rowid, title, excerpt, content_markdown)
  VALUES (new.id, new.title, new.excerpt, new.content_markdown);
END;

-- crawl history cho dashboard
CREATE TABLE crawl_runs (
  id                  INTEGER PRIMARY KEY,
  source_id           INTEGER NOT NULL REFERENCES sources(id),
  started_at          TEXT NOT NULL,
  finished_at         TEXT,
  status              TEXT NOT NULL,           -- 'running' | 'success' | 'failed' | 'cancelled'
  pages_fetched       INTEGER NOT NULL DEFAULT 0,
  articles_new        INTEGER NOT NULL DEFAULT 0,
  articles_updated    INTEGER NOT NULL DEFAULT 0,
  errors              INTEGER NOT NULL DEFAULT 0,
  error_message       TEXT
);

CREATE INDEX idx_runs_source_started ON crawl_runs(source_id, started_at DESC);

-- pragmas khi mở connection
PRAGMA journal_mode = WAL;       -- concurrent reads
PRAGMA synchronous  = NORMAL;
PRAGMA foreign_keys = ON;
PRAGMA busy_timeout = 5000;

Migration strategy

  • db/migrations/NNN_*.sql — fwd-only, chạy theo thứ tự, ghi version vào bảng schema_version.
  • Không dùng Alembic — overkill cho SQLite single-file. Tự viết 30 dòng migrate.py.
  • Seed sources từ configs/sources/*.yaml khi bắt đầu crawl run.

CLI hook

# start web UI (read-only browse, muốn phát triển thì tự chạy uvicorn)
star-crawl serve --host 127.0.0.1 --port 8000

# crawl chạy tách riêng qua CLI
star-crawl run grab_engineering
star-crawl run --all
Security: Mặc định bind 127.0.0.1. Nếu expose ra mạng phải thêm basic auth (env STAR_CRAWL_AUTH=user:pass) vì DB có thể chứa content có bản quyền.

Medium adapter (opt-in, experimental)

Medium robots.txt declare rõ User-Agent: ClaudeBot Disallow: /. Crawl chỉ khi user opt-in flag và hiểu rõ ToS.

Lưu ý ToS & pháp lý: Cả 3 phương án dưới cùng share rủi ro — Medium ToS prohibit API scraping; robots.txt explicit cấm AI crawlers. Code Freedium MIT-licensed nhưng không miễn trách nhiệm về content mà nó parse. Personal research & RAG cá nhân: rủi ro thấp. Redistribute / commercial training: rủi ro thật, không khuyến nghị.
#ApproachEffortReliabilityMaintenance
A Vendor medium-parser + gọi Medium GraphQL trực tiếp đề xuất 1–1.5d Cao (mình kiểm soát) TB — Medium đổi API là gãy
B Self-host Freedium docker-compose min profile 1.5d Cao Cao — 4 container + WARP
C Route URL qua freedium-mirror.cfd runtime 0.3d Thấp (mirror downtime) Thấp nhưng brittle

Adapter design (Option A)

# configs/sources/medium_graphql.yaml
name: medium_graphql
fetcher: medium_graphql            # custom adapter, vendored from Freedium
seed:
  strategy: hybrid
  feeds:                           # public feeds for discovery
    - "https://medium.com/feed/tag/distributed-systems"
    - "https://medium.com/feed/tag/machine-learning"
  follow_member_only: true
url_filter: '^https://medium\.com/.*-[a-f0-9]{8,}$'
extract:
  via_graphql: true                # post-id từ URL → GraphQL → HTML → trafilatura
  fallback_to_rss_content: true    # nếu GraphQL fail, dùng <content:encoded>
network:
  warp_proxy: optional             # cloudflare/cloudflared WARP
  rate_limit: { rps: 0.3, concurrency: 1 }
guards:
  require_flag: --i-understand-medium-tos    # default OFF

Open questions cần chốt

Trả lời 4 câu là scaffold P0 + P1 (Grab) ngay luôn.

  1. Mục đích cuối của dataset: research cá nhân / RAG riêng, hay training/sản phẩm thương mại? → quyết định Medium adapter có nên implement không.

  4. Phạm vi P1: chỉ Grab (validate pipeline), hay scaffold cho cả Uber/Gojek song song?
